AI Study Hub

A Smart Student Portal with AI-Powered Recommendations & Verified Storage

✦ Project Overview & Presentation ✦

Domain: EdTech Platform / Semantic Search / Content Evaluation

AI Engine: Gemini 2.5 Flash API

Storage Architecture: MongoDB GridFS (Permanent Cloud DB Storage)

Problem Statement

  • Quality Clutter: Traditional portals list study materials chronologically without evaluating content quality, forcing students to read poor notes.
  • Subject Mismatching: Students often upload files under incorrect subjects (e.g. Social Media Analytics notes under Deep Learning), cluttering search indexes.
  • File System Deletion: Cloud hosting containers (like Railway/Render) wipe local directories on redeploy, causing uploaded files to go missing.
The Objective: Build a smart repository that automatically parses PDFs, validates subject relevance, grades readability, highlights only the best notes, and preserves files permanently.

Key Platform Features

  • ⭐ Single "Best PDF" Constraint: Automatically groups documents by subject and awards the "Best PDF" badge to exactly one highest-rated file.
  • 🧠 Comparative AI Grading: Newly uploaded files are checked in-memory against the existing best file to determine if they deserve the recommendation star.
  • 🔐 OTP-verified Signup: Ensures student authenticity using email OTP registration powered by Brevo API.
  • 🔍 TF-IDF Semantic Search: Implements NLP search to return notes based on conceptual similarity.
  • 📊 Dynamic Data Dashboard: Offers visual analytics on upload counts, downloads, and branch distributions.

Technology Stack

Comparative AI Evaluation Workflow

1 File Upload: User uploads a PDF file under a specific subject.
2 In-Memory Fetch: Server fetches current best PDF and extracts its text.
3 Gemini Compare: Gemini compares both texts on readability & syllabus relevance.
4 Score Allocation: Better file gets a higher score; mismatched gets < 30.
5 Star Shifting: Badge automatically shifts to the new document if it scores higher.
Self-Healing Rule: If the Gemini API fails or no key is present, the server uses a keyword-based heuristic check to grade document relevance and assign score increments.

Database & GridFS Storage Architecture

  • Permanent Cloud Storage: Unlike local files, GridFS splits PDF binaries into small chunks and stores them in the remote database (MongoDB Atlas).
  • Container Independence: Cloud redeployments wipe local directories, but GridFS is never wiped. Files are streamed live on demand.
  • `fs.files` & `fs.chunks`: Split documents into chunks of 255KB each, ensuring fast uploads and memory-efficient downloads.
Zero-Downtime Reliability: Previews, admin previews, downloads, and deletion APIs are all integrated with GridFS and support smooth local disk fallbacks for legacy files.

User Interface & UX Highlights

  • 📊 Dynamic Stats: Chart.js charts render visual pie and bar distributions of study notes across various branches (CSE, IT, ECE, EEE).
  • ⭐ Interactive Star Badges: "⭐ Best PDF" badges render inline on search results, trending lists, and dashboard recommendations.
  • 📱 Responsive Dark Theme: Sleek glassmorphism and tailwind-free vanilla CSS styling optimized for mobile and desktop screens.
No Cache Lag: Search filters and page actions run asynchronously using JS Fetch AJAX API, ensuring smooth transitions without full-page reloads.

Conclusion & Future Enhancements

  • Collaborative Reading: Introduce online collaborative annotations and peer review notes.
  • Auto-Summaries: Leverage Gemini AI to auto-generate quick summary bullet points and flashcards for every note uploaded.
  • Audio Summaries: Generate text-to-speech audio recaps of the notes so students can listen on the go.
Summary: AI Study Hub successfully merges AI analysis, NLP semantic search, and secure cloud GridFS storage into a single, high-performing educational repository.
